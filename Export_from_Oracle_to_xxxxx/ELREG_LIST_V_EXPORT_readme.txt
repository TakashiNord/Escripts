
Условие:
 При создании типов полей необходимо выставлять очень много параметров (Имя, полное имя, единицу измерения, точность представления) и подтягивать источник значений. И когда создаётся большое количество типов на секцию шин, то вероятность ошибки очень велика, а инструмента для проверки не существует.  

Прошу помочь в создании скрипта. 

Задача: вывести для визуального просмотра поля
 Необходимо вывести в одну строку:
 имя распределительного устройства - имя секции шин - Краткое имя присоединения - Тип параметра - Полное имя типа параметра - Краткое имя типа параметра - Единица измерения - Точность представления - Источник значений - Параметр источника значений. 

Решение :
 запрос
Select ID, ID_PARENT ,ID_TYPE, NAME, ALIAS FROM OBJ_TREE \
    where ID_PARENT is null \
    -- where ID_TYPE in (select ID from SYS_OTYP where like '%OTYP_BUS_NETWORK%' ) " ;

сделали форматированный вывод с разделителем ;
* Одна выводит полное дерево Режимных, другая только те элементы , которые содержат тип "распределительное устройство".
Запуск, на компьютере где установлен клиент РСДУ (как пример) : mp.exe > d:\tree.txt

формат :
имя распределительного устройства ; имя секции шин | Краткое имя присоединения ; Тип параметра ; Полное имя типа параметра ; Краткое имя типа параметра ; Единица измерения ; Точность представления ; Источник значений ; Параметр источника значений.
